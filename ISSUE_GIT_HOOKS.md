# Git Hooks Missing Inter-Review Support

## Problem

Git hooks generated by psypi do not support `[inter-review:]` IDs in commit messages.

## Evidence

- 6,448 inter_reviews in database (6,443 with commit_hash)
- Recent psypi commits have `[task:]` and `[issue:]` but NO `[inter-review:]`
- Git hook blocks commits with `[inter-review:]` as invalid

## Root Cause

The psypi `setupHooks()` method generates `prepare-commit-msg` hooks that only validate `[task:]` and `[issue:]`, missing `[inter-review:]` validation.

**Current broken hook:**
```bash
if echo "$commit_msg" | grep -qE '\[(task|issue):'; then
```

**Required fix:**
```bash
if echo "$commit_msg" | grep -qE '\[(task|issue|inter-review):'; then
```

## Source Code

Location: `src/kernel/index.ts` lines 148-227

The `setupHooks()` method needs to be updated to include `inter-review` in the validation regex.

## Fix Required

1. Update `setupHooks()` method in `src/kernel/index.ts`:
   - Line 171: Change `'\\[(task|issue):'` to `'\\[(task|issue|inter-review):'`
   - Add issue resolution to post-commit hook
   - Add inter-review handling to post-commit hook

2. Run `psypi setup-hooks` to reinstall hooks

## Test

After fix:
```bash
# Should pass validation
git commit -m "Add feature [inter-review: bb43608c-d519-4fce-915d-9a89931a5008]"
```
